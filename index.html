Bienvenue sur mon portfolio. Voici une présentation de mon projet Zabbix :

Résumé du projet

  • Installation de Zabbix sur une VM avec PHP,Apache et une base de donnée MysQL
  • Mise en place d'un agent Zabbix sur mon hôte avec Docker pour faciliter la communication entre la VM et l'hôte.
  • Utilisation de Bash pour automatiser des tâches avec Cron, comme la gestion d'une IP dynamique.
  • Configuration d'un proxy sur l'agent pour améliorer la transmission des données.

Importance de ce projet

Ce projet met en avant :

  • L'importance d'une bonne configuration pour le monitoring système.
  • Les bénéfices d'automatisations dans des environnements dynamiques.
  • La sécurisation et l'optimisation des données transmises entre machines.
  • La nécessité de bien gérer l'environnement système, y compris :
    • Allocation manuelle de mémoire avec modification des partitions.
    • Changement du mode réseau de la VM de NAT à bridge pour éviter des conflits réseau.
    • Résolution de conflits de ports entre agents et services.
    • Gestion des versions de PHP et different service pour éviter les conflits entre anciennes et nouvelles versions.

Scripts utilisés

  • Script pour l'agent Zabbix : Met à jour automatiquement l'adresse IP dynamique et redémarre les services nécessaires.
  • Script pour le serveur : Gère les mises à jour d'IP pour Apache, PHP et le serveur Zabbix dans un fichier de configuration unique.
